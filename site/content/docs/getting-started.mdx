This guide stands up a minimal Durable Streams HTTP server using the in-memory storage backend bundled with `@streamsy/core`. It is the fastest way to try the protocol locally before moving to a durable backend such as SQLite or Cloudflare Durable Objects.

## Install

Streamsy targets the Bun runtime. Add the core package:

```bash
bun add @streamsy/core
```

## Compose the server

A Streamsy application usually composes three pieces:

1. a `StreamFactory` from a storage package (here, the in-memory factory from `@streamsy/core`);
2. a protocol facade from `createStreamProtocol({ storage: { factory } })`;
3. an HTTP facade from `createHttpHandler({ protocol, pathPrefix })` that serves the Durable Streams HTTP protocol.

```ts
import { createHttpHandler, createMemoryStreamFactory, createStreamProtocol } from "@streamsy/core";

const factory = createMemoryStreamFactory();
const protocol = createStreamProtocol({ storage: { factory } });
const handler = createHttpHandler({ protocol, pathPrefix: "/" });

Bun.serve({ fetch: (request) => handler.fetch(request) });
```

Run it with Bun and the server will speak the Durable Streams HTTP protocol on the configured path prefix.

## Typed layers

`@streamsy/json` and `@streamsy/state` wrap a `StreamProtocolFactory` with typed facades:

```ts
import { createJsonProtocol } from "@streamsy/json";
import { createDurableStateProtocol } from "@streamsy/state";

// Typed JSON streams: values validated through a JsonCodec or Standard Schema.
const json = createJsonProtocol(protocol, userCodec);
const created = await json.create("users", { initialMessage: { id: "u1", name: "Alice" } });

// Durable State streams: typed change/control messages over collections.
const durable = createDurableStateProtocol(protocol, {
  users: { type: "user", schema: userCodec, primaryKey: "id" },
});
```

## Choose a durable backend

The in-memory factory is ideal for development, examples, and conformance testing. For persistence, swap the factory for a durable storage adapter:

- **`@streamsy/storage-sqlite`** — Bun `bun:sqlite` persistence with automatic migrations, mutation locking, live-read notification, and expiry. Requires the Bun runtime.
- **`@streamsy/storage-durable-object`** — Cloudflare Durable Object storage with SQLite-backed persistence, long-polling, SSE, and TTL/expiry handling.

The protocol and HTTP facades stay the same; only the factory passed to `createStreamProtocol` changes.
